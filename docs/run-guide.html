วิธี Run ระบบ ยินผัน ฟุตบอล อคาเดมี

โปรเจกต์นี้เป็น Next.js แอปเดียวที่ทำหน้าที่ทั้งฝั่งแสดงผล (frontend) และฝั่งประมวลผล/ฐานข้อมูล (backend) ในตัวเดียวกัน สิ่งเดียวที่รันแยกออกไปคือฐานข้อมูล PostgreSQL ผ่าน Docker — ไม่มีคำสั่ง "run backend" กับ "run frontend" แยกกันสองอัน

สรุป: รันแค่ 2 อย่าง — Docker (ฐานข้อมูล) กับ npm run dev (ทั้งเว็บและ backend อยู่ในนั้นแล้ว)

ก่อนเริ่ม

เครื่องมือที่ต้องติดตั้งไว้ล่วงหน้า

เครื่องมือใช้ทำอะไรเช็คว่ามีแล้วหรือยัง
Node.jsรันโปรเจกต์ Next.js และคำสั่ง npmnode -v
Docker Desktopรันฐานข้อมูล PostgreSQL แบบไม่ต้องติดตั้งเองdocker -v และเปิดแอป Docker Desktop ทิ้งไว้
โฟลเดอร์โปรเจกต์ซอร์สโค้ดทั้งหมดของระบบD:\football-academy
เปิด Docker Desktop ทิ้งไว้เสมอ ก่อนรันคำสั่งใดๆ ที่เกี่ยวกับฐานข้อมูล — ถ้า Docker ไม่ทำงาน คำสั่งที่เกี่ยวกับ db จะ error ทันที
ทำครั้งเดียว

ติดตั้งระบบครั้งแรก

ทำตามลำดับนี้เฉพาะครั้งแรกที่ดึงโปรเจกต์มา หรือครั้งแรกที่สอนนักเรียนตั้งเครื่องใหม่

1

เปิด Terminal ที่โฟลเดอร์โปรเจกต์

เปิด PowerShell หรือ Terminal แล้วเข้าไปที่โฟลเดอร์ football-academy

# ตัวอย่างถ้าโปรเจกต์อยู่ที่ D:\football-academy
cd D:\football-academy
2

ติดตั้ง dependencies

ดาวน์โหลดไลบรารีทั้งหมดที่โปรเจกต์ต้องใช้ (ครั้งแรกจะใช้เวลาสักครู่)

npm install
3

เปิดฐานข้อมูล PostgreSQL (Docker)

คำสั่งนี้จะสร้างและเปิดคอนเทนเนอร์ฐานข้อมูลเบื้องหลัง (รันครั้งเดียว ครั้งต่อไปแค่เปิดใหม่ก็พอ)

npm run db:up
4

สร้างตารางในฐานข้อมูล (migrate)

สร้างโครงสร้างตารางทั้งหมดตาม schema ของระบบ (users, courses, bookings ฯลฯ)

npm run db:migrate
5

ใส่ข้อมูลตัวอย่าง (seed)

สร้างบัญชีทดสอบ + คอร์ส/รุ่น/บริการคลินิกตัวอย่าง เพื่อให้มีข้อมูลไว้ทดสอบทันที

npm run db:seed
ทำ 3 ขั้นตอนสุดท้าย (3–5) ตามลำดับเป๊ะๆ — ถ้า seed ก่อน migrate จะ error เพราะยังไม่มีตารางให้ใส่ข้อมูล
ทำทุกครั้งที่จะสอน

รันระบบประจำวัน

หลังติดตั้งครั้งแรกแล้ว วันถัดไปทำแค่ 2 ขั้นตอนนี้พอ

1

เปิดฐานข้อมูล (ถ้ายังไม่ได้เปิด)

ถ้า Docker container เคยรันไว้แล้วและเครื่องไม่ได้รีสตาร์ท ข้ามขั้นตอนนี้ได้เลย

npm run db:up
2

เปิด dev server (frontend + backend พร้อมกัน)

คำสั่งเดียวนี้รันทั้งเว็บและ backend logic ทั้งหมด แล้วเปิดเบราว์เซอร์ไปที่ http://localhost:3000

npm run dev
ปล่อย Terminal หน้าต่างนี้ทิ้งไว้ตลอดตอนสอน — ถ้าปิดหรือกด Ctrl+C เว็บจะหยุดทำงานทันที
อ้างอิงเร็ว

สรุปคำสั่งทั้งหมด

คำสั่งทำอะไรใช้บ่อยแค่ไหน
npm installติดตั้ง dependenciesครั้งแรก / เมื่อมีไลบรารีใหม่
npm run db:upเปิดฐานข้อมูล Postgres ใน Dockerทุกครั้งก่อนเริ่มงาน
npm run db:downปิดฐานข้อมูลเมื่อเลิกใช้เครื่อง
npm run db:migrateอัปเดตโครงสร้างตารางตาม schemaเมื่อ schema เปลี่ยน
npm run db:seedใส่ข้อมูลตัวอย่าง/บัญชีทดสอบครั้งแรก / เมื่อล้างข้อมูลใหม่
npm run devรันเว็บ+backend สำหรับพัฒนา (แก้โค้ดแล้วเห็นผลทันที)ทุกครั้งที่สอน/พัฒนา
npm run buildตรวจสอบโค้ดและ build เวอร์ชันจริงสำหรับ deployก่อนขึ้นเว็บจริง
npm run lintตรวจสอบคุณภาพโค้ดก่อน commit
เมื่อเลิกใช้งาน

วิธีหยุดระบบ

1

หยุด dev server

กด Ctrl + C ใน Terminal ที่รัน npm run dev อยู่

2

ปิดฐานข้อมูล (ไม่บังคับ)

ถ้าจะไม่ใช้อีกสักพัก ปิด container ประหยัดทรัพยากรเครื่องได้ — ข้อมูลยังอยู่ครบ เปิดใหม่ด้วย npm run db:up ได้เหมือนเดิม

npm run db:down
เจอปัญหา

แก้ปัญหาที่พบบ่อย

Port 3000 is in use

มีโปรแกรมอื่นใช้พอร์ต 3000 อยู่ — Next.js จะเปลี่ยนไปใช้พอร์ตอื่นให้อัตโนมัติและบอกในหน้าจอ (เช่น 3001) ให้เปิดพอร์ตนั้นแทน หรือปิดโปรแกรมที่ใช้พอร์ต 3000 อยู่ก่อน

Can't reach database server / connection refused

Docker ยังไม่ได้เปิด หรือ container ฐานข้อมูลยังไม่รัน — เปิดแอป Docker Desktop แล้วรัน npm run db:up ใหม่

Table does not exist

ยังไม่ได้รัน npm run db:migrate — ต้องสร้างตารางก่อนถึงจะใช้งานได้

Login ไม่ผ่านทั้งที่ใส่รหัสถูก

ยังไม่ได้รัน npm run db:seed จึงยังไม่มีบัญชีทดสอบในฐานข้อมูล — ดูรายชื่อบัญชีด้านล่าง

สำหรับสอนนักเรียน

บัญชีทดสอบหลัง seed

ใช้ล็อกอินที่ /login เพื่อสาธิตแต่ละมุมมองของระบบให้นักเรียนดู
