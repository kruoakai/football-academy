คู่มือ Deploy

ติดตั้งระบบขึ้น Ubuntu Server จริง — 3 วิธี เลือกได้ตามความถนัด

คู่มือนี้ต่อยอดจาก run-guide.html (รันบนเครื่อง dev) โดยพาไปติดตั้งระบบยินผัน ฟุตบอล อคาเดมี บนเซิร์ฟเวอร์ Ubuntu จริงแบบ production ครบทั้งฐานข้อมูล, HTTPS, และการอัปเดตระบบภายหลัง เลือกได้ 3 วิธี: ใช้ Docker จัดการเอง, ใช้ Docker ร่วมกับ CI/CD อัตโนมัติ, หรือไม่ใช้ Docker เลย

ก่อนเริ่ม — เตรียมเซิร์ฟเวอร์ (ทำครั้งเดียว ใช้ได้ทั้ง 3 วิธี)

Prerequisites

ไม่ว่าจะเลือกวิธีไหน ต้องเตรียมเซิร์ฟเวอร์ 5 ขั้นตอนนี้ก่อนเสมอ

1

เช่า/เตรียมเซิร์ฟเวอร์ Ubuntu

แนะนำ Ubuntu 22.04 หรือ 24.04 LTS, RAM อย่างน้อย 2GB (แนะนำ 4GB ขึ้นไปถ้ารัน Docker), พื้นที่ดิสก์อย่างน้อย 20GB

2

สร้างผู้ใช้ที่ไม่ใช่ root

ห้าม deploy ด้วย root โดยตรง — สร้างผู้ใช้ใหม่แล้วให้สิทธิ์ sudo

sudo adduser deploy
sudo usermod -aG sudo deploy
# ล็อกเอาต์แล้วล็อกอินใหม่ด้วยผู้ใช้ deploy ก่อนทำขั้นตอนถัดไป
3

ชี้โดเมนมาที่เซิร์ฟเวอร์

ไปที่ผู้ให้บริการโดเมน สร้าง DNS A record ชี้ yourdomain.com และ www.yourdomain.com ไปที่ IP ของเซิร์ฟเวอร์ รอ DNS propagate (ปกติไม่กี่นาทีถึง 1 ชั่วโมง)

4

ตั้งค่า Firewall (UFW)

เปิดเฉพาะพอร์ตที่จำเป็น — SSH, HTTP, HTTPS

sudo apt update && sudo apt upgrade -y
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
5

ติดตั้ง Nginx + Certbot

ใช้ Nginx เป็น reverse proxy หน้าแอป และ Certbot ขอใบรับรอง SSL ฟรีจาก Let's Encrypt — ใช้ตัวเดียวกันได้ทั้ง 3 วิธี เพราะสุดท้ายแอปฟังอยู่ที่ 127.0.0.1:3000 เหมือนกันหมด

sudo apt install -y nginx certbot python3-certbot-nginx
ไฟล์ตัวอย่าง Nginx config อยู่ที่ deploy/nginx.conf.example ในโปรเจกต์ — ใช้ตัวเดียวกันได้ในทั้ง 3 วิธี ขั้นตอนตั้งค่าอยู่ในแต่ละแท็บด้านล่าง

วิธีติดตั้ง — เลือก 1 ใน 3

Deploy methods

เหมาะกับ: ทีมที่คุ้นเคย Docker อยู่แล้ว อยากแยกสภาพแวดล้อมของแอปออกจากเครื่องจริง
Build image บนเซิร์ฟเวอร์เอง แล้วรันด้วย Docker Compose — อัปเดตครั้งถัดไปรันคำสั่งเองไม่กี่บรรทัด

1

ติดตั้ง Docker Engine + Compose plugin

ใช้ repo ทางการของ Docker (ไม่ใช้ snap install docker เพราะจัดการ volume/permission ยุ่งยากกว่า)

sudo apt install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo usermod -aG docker $USER
# ล็อกเอาต์แล้วล็อกอินใหม่ เพื่อให้สิทธิ์กลุ่ม docker มีผล
2

Clone โปรเจกต์ลงเซิร์ฟเวอร์

git clone https://github.com/YOUR-ORG/football-academy.git /var/www/football-academy
cd /var/www/football-academy
3

สร้างไฟล์ค่าจริง .env.production

cp .env.example .env.production
nano .env.production
สำคัญ: ใน .env.production ให้ DATABASE_URL ใช้ host ชื่อ db ไม่ใช่ localhost (เพราะรันอยู่ใน docker network เดียวกัน) เช่น postgresql://football_academy:รหัสผ่านจริง@db:5432/football_academy?schema=public และสุ่ม AUTH_SECRET ใหม่เสมอ — ห้ามก็อปค่าจาก dev มาใช้: openssl rand -base64 32
4

Build image และรันฐานข้อมูล + migration

ใช้ docker-compose.prod.yml ที่มากับโปรเจกต์ — มี 3 services: db, app, และ migrate (job รันครั้งเดียวสำหรับ prisma migrate deploy)

docker compose -f docker-compose.prod.yml build
docker compose -f docker-compose.prod.yml up -d db
docker compose -f docker-compose.prod.yml run --rm migrate
docker compose -f docker-compose.prod.yml up -d app
ห้ามรัน prisma db seed (มีบัญชีทดสอบรหัสผ่านที่รู้กันทั่วไป) บนฐานข้อมูลจริง — ให้สร้างบัญชีแอดมินคนแรกด้วยสคริปต์แยกหรือ Prisma Studio ผ่าน SSH tunnel แทน
5

ตั้งค่า Nginx reverse proxy + SSL

sudo cp deploy/nginx.conf.example /etc/nginx/sites-available/football-academy
sudo nano /etc/nginx/sites-available/football-academy
# แก้ yourdomain.com เป็นโดเมนจริงในไฟล์นี้ก่อน save
sudo ln -s /etc/nginx/sites-available/football-academy /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
6

ตรวจสอบว่าใช้งานได้จริง

docker compose -f docker-compose.prod.yml ps
docker compose -f docker-compose.prod.yml logs -f app

แล้วเปิด https://yourdomain.com ในเบราว์เซอร์ ควรเห็นหน้าแรกของเว็บและมีแม่กุญแจ HTTPS

7

อัปเดตระบบครั้งถัดไป (มีโค้ดใหม่)

cd /var/www/football-academy
git pull
docker compose -f docker-compose.prod.yml build app migrate
docker compose -f docker-compose.prod.yml run --rm migrate
docker compose -f docker-compose.prod.yml up -d app
docker image prune -f

เหมาะกับ: ทีมที่ deploy บ่อย หรือมีหลายคนดูแลร่วมกัน
ต่อยอดจากวิธีที่ 1 — เพิ่ม GitHub Actions ให้ build image และ deploy อัตโนมัติทุกครั้งที่ push โค้ดขึ้น branch main

1

เตรียมเซิร์ฟเวอร์เหมือนวิธีที่ 1 (ข้อ 1-3)

ติดตั้ง Docker, clone โปรเจกต์ไปที่ /var/www/football-academy, สร้าง .env.production ให้ครบ — แต่ยังไม่ต้อง build เอง เพราะ GitHub Actions จะ build ให้

2

สร้าง SSH key คู่ใหม่สำหรับ CI (แยกจาก key ส่วนตัว)

ssh-keygen -t ed25519 -C "deploy-bot" -f ~/.ssh/deploy_key -N ""
cat ~/.ssh/deploy_key.pub >> ~/.ssh/authorized_keys
cat ~/.ssh/deploy_key
# คัดลอกเนื้อหา private key ทั้งหมด (รวม BEGIN/END) ไปใส่ GitHub secret ในขั้นตอนถัดไป
3

ตั้งค่า GitHub repo secrets

ไปที่ GitHub repo → Settings → Secrets and variables → Actions → New repository secret

Secretค่า
SSH_HOSTIP หรือโดเมนของเซิร์ฟเวอร์
SSH_USERชื่อผู้ใช้ที่สร้างไว้ (เช่น deploy)
SSH_PRIVATE_KEYเนื้อหาไฟล์ ~/.ssh/deploy_key ทั้งหมด
DEPLOY_PATH/var/www/football-academy
4

เปิด GHCR package เป็น public (ครั้งแรกหลัง push image แรก)

Workflow จะ push image ไปที่ GitHub Container Registry (ghcr.io) — ให้เซิร์ฟเวอร์ดึง image ได้โดยไม่ต้อง login ซับซ้อน ให้เปิด package เป็น public ครั้งเดียว: GitHub → หน้า repo → แท็บ Packages → เลือก package → Package settings → เลื่อนไป Danger Zone → Change visibility → Public

ถ้าไม่อยากเปิด public ก็ทำได้ — สร้าง Personal Access Token สิทธิ์ read:packages แล้วรัน docker login ghcr.io -u USERNAME บนเซิร์ฟเวอร์ครั้งเดียวแทน
5

Push โค้ดขึ้น branch main

ไฟล์ .github/workflows/deploy.yml มากับโปรเจกต์แล้ว — เมื่อ push ขึ้น main จะรันอัตโนมัติ: build image → push ขึ้น ghcr.io → SSH เข้าเซิร์ฟเวอร์ → git pull → pull image ใหม่ → รัน migration → restart แอป

git push origin main
6

ตรวจสอบผลใน GitHub Actions

ไปที่แท็บ Actions ของ repo — ควรเห็น job build-and-push และ deploy ขึ้นสีเขียวทั้งคู่ ถ้า deploy แดง ให้ดู log ใน step "SSH and redeploy" เพื่อหาสาเหตุ

7

Rollback ถ้า deploy ล่าสุดมีปัญหา

ทุก image มี tag เป็น git commit SHA ด้วย — ย้อนกลับไปเวอร์ชันก่อนหน้าได้ทันทีโดยไม่ต้อง revert โค้ด

ssh deploy@yourdomain.com
cd /var/www/football-academy
export APP_IMAGE=ghcr.io/YOUR-ORG/football-academy:<sha เก่าที่ต้องการ>
docker compose -f docker-compose.prod.yml up -d app

เหมาะกับ: เซิร์ฟเวอร์ทรัพยากรจำกัด หรืออยากตั้งค่าให้เร็ว/ง่ายที่สุด
ติดตั้ง Node.js และ PostgreSQL ลงเครื่องโดยตรง ไม่ผ่าน Docker เลย ใช้ PM2 คุมโปรเซสให้รันตลอดเวลา

1

ติดตั้ง Node.js 20 LTS

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v
2

ติดตั้ง PostgreSQL และสร้างฐานข้อมูล

sudo apt install -y postgresql postgresql-contrib
sudo -u postgres psql -c "CREATE USER football_academy WITH PASSWORD 'รหัสผ่านจริง';"
sudo -u postgres psql -c "CREATE DATABASE football_academy OWNER football_academy;"
PostgreSQL ที่ติดตั้งผ่าน apt จะฟังอยู่ที่ localhost เท่านั้นโดย default (ไม่เปิดออก internet) อยู่แล้ว — ไม่ต้องแก้ postgresql.conf เพิ่ม
3

ติดตั้ง PM2 (ตัวคุมโปรเซสให้รันตลอดเวลา)

sudo npm install -g pm2
4

Clone โปรเจกต์ ตั้งค่า .env แล้ว build

git clone https://github.com/YOUR-ORG/football-academy.git /var/www/football-academy
cd /var/www/football-academy
cp .env.example .env
nano .env
# DATABASE_URL ใช้ host เป็น localhost ได้เลย (Postgres รันบนเครื่องเดียวกัน)
# AUTH_SECRET ต้องสุ่มใหม่: openssl rand -base64 32
npm ci

npm run build
5

รันแอปด้วย PM2

ใช้ config ที่มากับโปรเจกต์ deploy/ecosystem.config.js

pm2 start deploy/ecosystem.config.js
pm2 save
pm2 startup
# ทำตามคำสั่ง sudo อีก 1 บรรทัดที่ pm2 พิมพ์ออกมา เพื่อให้แอปเริ่มอัตโนมัติทุกครั้งที่เซิร์ฟเวอร์รีบูต
6

ตั้งค่า Nginx reverse proxy + SSL

sudo cp deploy/nginx.conf.example /etc/nginx/sites-available/football-academy
sudo nano /etc/nginx/sites-available/football-academy

sudo ln -s /etc/nginx/sites-available/football-academy /etc/nginx/sites-enabled/

7

อัปเดตระบบครั้งถัดไป (มีโค้ดใหม่)

ตัวแปรสภาพแวดล้อมที่ต้องตั้งค่าจริง

Environment variables

อ้างอิงจาก .env.example ในโปรเจกต์ — ค่าที่มีเครื่องหมาย * ต้องเปลี่ยนจากค่า dev เสมอ

ตัวแปรคำอธิบาย
DATABASE_URL *connection string ของ Postgres จริง (host เป็น db ถ้าใช้ Docker, localhost ถ้าไม่ใช้)
AUTH_SECRET *สุ่มใหม่ด้วย openssl rand -base64 32 ห้ามใช้ค่าจาก dev
NEXT_PUBLIC_SITE_URL *โดเมนจริง เช่น https://yourdomain.com (ใช้ทำ sitemap/SEO/absolute URL)
LINE_CHANNEL_ACCESS_TOKENจาก LINE Official Account Manager — เว้นว่างได้จนกว่าจะเชื่อมต่อจริง
LINE_CHANNEL_SECRETคู่กับด้านบน
NEXT_PUBLIC_LINE_OA_IDLINE Official Account ID
RESEND_API_KEYจาก resend.com หลังยืนยัน sending domain แล้ว — เว้นว่างได้จนกว่าจะเชื่อมต่อจริง

เช็คลิสต์ก่อนเปิดใช้งานจริง

Security
ข้อมูลบาดเจ็บ/สุขภาพ (injury_records): ตาม CLAUDE.md ของโปรเจกต์ ตารางนี้ยังไม่ผ่านการตรวจสอบด้านกฎหมาย PDPA ควรปรึกษาผู้เชี่ยวชาญด้าน PDPA/ข้อมูลสุขภาพก่อนเปิดใช้งานกับข้อมูลผู้ใช้จริงบน production
สุ่ม AUTH_SECRET ใหม่บน productionห้ามใช้ค่าเดียวกับ dev — ถ้ารั่วเท่ากับปลอมแซง session ผู้ใช้ทุกคนได้
บังคับ HTTPS ทุกหน้าCertbot ตั้งค่า redirect HTTP → HTTPS ให้อัตโนมัติเมื่อรันเสร็จ
ไม่เปิดพอร์ต Postgres ออก internetทั้งวิธี Docker (ไม่ map 5432 ออก) และไม่ใช้ Docker (Postgres ฟัง localhost อยู่แล้ว) เป็นค่า default ที่ปลอดภัยแล้ว — อย่าไปแก้ให้เปิดออก
ห้ามรัน prisma db seed บนฐานข้อมูลจริงมีบัญชีทดสอบรหัสผ่านที่รู้กันทั่วไป (Admin123! ฯลฯ) — สร้างบัญชีแอดมินจริงด้วยมือแทน
ตั้ง backup ฐานข้อมูลสม่ำเสมอดูหัวข้อถัดไป — ควรส่งไฟล์ backup ไปเก็บนอกเซิร์ฟเวอร์ด้วย ไม่ใช่เก็บที่เดียวกับฐานข้อมูลจริง

สำรอง / กู้คืนฐานข้อมูล

Backup & restore
D

วิธี Docker — สำรองข้อมูล

docker compose -f docker-compose.prod.yml exec db pg_dump -U football_academy football_academy | gzip > backup_$(date +%F).sql.gz

วิธี Docker — กู้คืนข้อมูล

gunzip -c backup_2026-07-10.sql.gz | docker compose -f docker-compose.prod.yml exec -T db psql -U football_academy football_academy
N

วิธีไม่ใช้ Docker — สำรอง/กู้คืนข้อมูล

pg_dump -U football_academy football_academy | gzip > backup_$(date +%F).sql.gz
# กู้คืน:
gunzip -c backup_2026-07-10.sql.gz | psql -U football_academy football_academy
แนะนำตั้ง cron job รัน backup ทุกวัน แล้วอัปโหลดไฟล์ไปเก็บที่อื่น (เช่น S3-compatible storage) — เก็บ backup ไว้เครื่องเดียวกับฐานข้อมูลจริงอย่างเดียวไม่ปลอดภัยพอ

Troubleshooting
อาการสาเหตุ / วิธีแก้
Nginx ขึ้น 502 Bad Gatewayแอปยังไม่ทำงานที่พอร์ต 3000 — เช็ค docker compose logs app หรือ pm2 logs football-academy
แก้ schema แล้วเจอ error "does not exist" หลัง deployต้อง restart แอปหลัง prisma generate/migrate deploy ใหม่เสมอ — โปรเซสเก่าที่รันอยู่จะไม่โหลด client ใหม่ให้เองแม้ไฟล์จะอัปเดตแล้ว (รีสตาร์ท container หรือ pm2 restart)
Port 3000 ถูกใช้อยู่แล้วsudo lsof -i :3000 หาโปรเซสเก่าแล้วปิดก่อนเริ่มใหม่
ใบรับรอง SSL ใกล้หมดอายุCertbot ตั้ง systemd timer ต่ออายุอัตโนมัติให้แล้ว เช็คด้วย sudo certbot renew --dry-run
docker compose run migrate error P1001 (reach database)ฐานข้อมูลยัง healthy ไม่ครบ — เช็คด้วย docker compose ps ว่า db ขึ้นสถานะ healthy แล้วหรือยัง
ดึง image จาก GHCR บนเซิร์ฟเวอร์ไม่ได้ (unauthorized)Package ยังเป็น private — ตั้งเป็น public หรือ docker login ghcr.io ด้วย Personal Access Token ก่อน
